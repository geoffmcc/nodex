# Threat Model

| Threat | Asset | Entry point | Trust boundary | Impact | Current controls / evidence | Residual risk |
|---|---|---|---|---|---|---|
| Malicious local strings in Proxmox names/comments/logs | Operator terminal, CI logs, automation parsers | Provider responses rendered by CLI | Proxmox endpoint -> local stdout/stderr | Terminal control injection, log forging, misleading output | SEC-001 sanitizes JSON/YAML string values; SEC-003 wraps direct CLI writes; table/error paths already sanitize | Low: consumers may intentionally decode escaped JSON strings in their own tools |
| Credential leakage in output/errors | API tokens, passwords, credential refs | Errors, logs, structured output, tests | Local secret stores/env -> CLI sinks | Token disclosure | `redact.Secret`, `Redactable`, regex redaction, top-level error redaction, output writer wrappers, fake fixture scanner review | Low: compromised local machine can still read env/keyring/file backends |
| MITM/hostile endpoint/redirect | Proxmox credentials and data | HTTP transport and redirects | Local client -> network endpoint | Credential forwarding or downgrade | HTTPS-only endpoint validation, TLS 1.2+, custom CA preserves hostname validation, cross-origin/downgrade redirect block, SEC-002 redirect cap | Live CA/PKI behavior requires operator validation against real endpoint |
| Unsafe non-idempotent retry/unknown outcome | VMs, containers, storage, cluster tasks | POST/PUT/DELETE mutations | CLI/provider -> Proxmox task system | Duplicate mutation, hidden partial completion | `DoMutation` no retry; task unknown-outcome types/tests; polling uses contexts/timeouts | Real Proxmox task disappearance/HA edge cases require live validation plan |
| Wrong target/wrong resource type | Guests, storage, nodes | CLI target parsing and provider methods | User input -> Proxmox path | Wrong VM/CT/node/storage mutation | Explicit node/vmid parsing, separate qemu/lxc client methods, safety tiers, path escaping, operation inventory | No live cluster identity revalidation before every mutation; documented residual risk |
| Noninteractive unsafe confirmation | Proxmox state | Mutation flags | Automation -> destructive operations | Data loss or disruption | Safety tiers require `--yes`, `--force`, typed target, `--expert`; noninteractive fails closed when prompt required | Operator can deliberately pass confirming flags |
| Filesystem path attacks | Config/credentials/download destinations | Local file writes | User input -> filesystem | Secret overwrite, symlink writes, traversal | Atomic writes, restricted dirs/files, credential name validation, download path validation/non-regular rejection | Windows ACL equivalence remains platform-dependent |
| CI/supply chain compromise | Build/test/release integrity | GitHub Actions, Go modules | GitHub runner/network -> project | Compromised artifacts/tests | Read-only workflow permissions, SHA-pinned actions, pinned Go tool installs, `go mod verify`, govulncheck/gosec/staticcheck | Go install tool checksum trust is Go module ecosystem trust |
